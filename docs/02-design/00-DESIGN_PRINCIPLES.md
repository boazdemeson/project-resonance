# Design Principles

## Purpose

This document defines the fundamental user experience principles that guide the design of the platform.

These principles are intentionally long-lived.

Features, layouts, and visual styles may evolve, but these principles should remain consistent as the product grows.

When design decisions conflict, these principles take precedence.

---

# 1. Creativity Comes First

The platform exists to support creativity.

Every interaction should help musicians create, continue, or complete a creative Session.

Administrative tasks should never interrupt the creative process.

> If a design forces users to think about software instead of music, redesign it.

---

# 2. Sessions Are the Center

The Session is the primary experience of the platform.

Everything that contributes to creating music belongs to a Session.

The interface should consistently reinforce this mental model.

Users should never wonder where creative information belongs.

---

# 3. Capture Before Organize

Ideas are more valuable than metadata.

The platform should make capturing creative work effortless.

Organization should always be secondary to creation.

Users can organize later.

They cannot recover a forgotten idea.

---

# 4. Preserve Creative History

Creative work should never feel fragile.

Users should feel confident experimenting because previous work remains available.

The platform should encourage exploration rather than fear of making mistakes.

Creative history is part of the product.

---

# 5. Reduce Mental Effort

Creating music is already mentally demanding.

The software should reduce unnecessary thinking.

Users should spend their attention on creativity rather than navigation, settings, or workflow management.

Whenever possible, the platform should make reasonable decisions on behalf of the user.

---

# 6. Solo by Default

The platform should feel complete for a musician working alone.

Collaboration should extend existing workflows without replacing them.

Users should never feel forced into team-oriented experiences.

---

# 7. Calm Interfaces Create Better Music

The interface should support focus.

Every element on screen should have a clear purpose.

Remove unnecessary complexity.

Remove unnecessary distraction.

A calm interface creates more room for creativity.

---

# Design Review Questions

Every new screen, feature, or interaction should answer "Yes" to these questions.

- Does this help musicians create?
- Does this reduce friction?
- Does this reduce mental effort?
- Does this preserve creative history?
- Does this reinforce the Session as the center of the experience?
- Could this be simpler?

If the answer to any question is "No", the design should be reconsidered.

---

# UX Success Metrics

The experience is successful when:

- Users can create a Session in under 10 seconds.
- Users can capture their first idea in under 30 seconds.
- Users can resume any active Session in one click.
- Users never lose creative work.
- Users always know where they are.
- Users always understand what to do next.

---

# Decisions

- The Session is the center of the product.
- Creation is prioritized over management.
- Organization supports creativity.
- Collaboration is optional.
- Creative history is preserved.
- Simplicity is preferred over feature density.
